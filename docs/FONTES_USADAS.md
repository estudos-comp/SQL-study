## Observações de normalização

Algumas grafias e exemplos foram normalizados para a forma canônica:

- `Foreing Key` → `FOREIGN KEY`.
- `REFERENCE` isolado → `REFERENCES` no comando SQL.
- `INTERSEC` → `INTERSECT`.
- `BEGIIN TRANSACTION` → `BEGIN TRANSACTION`.
- Aspas tipográficas foram substituídas por aspas simples comuns em blocos SQL.
- Nomes com hífen dos slides, como `Trabalha-em`, foram adaptados para `trabalha_em` em exemplos novos.

As diferenças entre SGBDs foram preservadas quando eram parte do conteúdo, principalmente em autoincremento, criação de usuários, procedures, functions e triggers.

## Modelo usado nos exercícios

Considere, quando necessário, este esquema simplificado:

```sql
aluno(mat, nomea, curso, turma, depto, data_nasc)
disc(d, nomed, carga_h)
aluno_paga(mat, d, turma, data_pagamento)
prof(matricula, nome, categoria, salario)
depto(cod_depto, nome_depto)
empregado(id_empreg, nome, cod_depto, limite_credito, fone, id_gerente, salario)
pedido(num_pedido, cod_cliente, cod_vendedor, data_pedido)
item_pedido(num_pedido, cod_produto, qtd)
produto(cod_produto, descricao, valor)
```

---

## 1. Listar nome e curso de todos os alunos

```sql
SELECT nomea, curso
FROM aluno;
```

---

## 2. Listar todas as disciplinas sem repetir carga horária

```sql
SELECT DISTINCT carga_h
FROM disc;
```

---

## 3. Listar alunos do curso de Ciência da Computação

```sql
SELECT mat, nomea
FROM aluno
WHERE curso = 'Ciência da Computação';
```

---

## 4. Listar empregados com salário entre 1000 e 2000

```sql
SELECT *
FROM empregado
WHERE salario BETWEEN 1000 AND 2000;
```

---

## 5. Listar disciplinas cujo nome contém “Banco”

```sql
SELECT *
FROM disc
WHERE nomed LIKE '%Banco%';
```

---

## 6. Listar vendedores ou empregados sem gerente

```sql
SELECT nome, id_gerente
FROM empregado
WHERE id_gerente IS NULL;
```

---

## 7. Listar alunos que pagam disciplinas com carga horária diferente de 60h

```sql
SELECT a.nomea
FROM aluno a
INNER JOIN aluno_paga ap ON ap.mat = a.mat
WHERE ap.d IN (
    SELECT d.d
    FROM disc d
    WHERE d.carga_h <> 60
);
```

---

## 8. Listar nome do aluno e nome da disciplina paga

```sql
SELECT a.nomea, d.nomed
FROM aluno a
INNER JOIN aluno_paga ap ON ap.mat = a.mat
INNER JOIN disc d ON d.d = ap.d;
```

---

## 9. Contar quantas disciplinas cada aluno paga

```sql
SELECT ap.mat, COUNT(*) AS total_disciplinas
FROM aluno_paga ap
GROUP BY ap.mat;
```

---

## 10. Contar disciplinas por aluno, mas só alunos com mais de 2 disciplinas

```sql
SELECT ap.mat, COUNT(*) AS total_disciplinas
FROM aluno_paga ap
GROUP BY ap.mat
HAVING COUNT(*) > 2;
```

---

## 11. Listar departamentos com empregados ganhando mais de 1000, apenas se houver mais de 5 empregados

```sql
SELECT cod_depto, COUNT(*) AS total_empregados
FROM empregado
WHERE salario > 1000
GROUP BY cod_depto
HAVING COUNT(*) > 5;
```

---

## 12. Listar empregados com salário maior que a média

```sql
SELECT nome, salario
FROM empregado
WHERE salario > (
    SELECT AVG(salario)
    FROM empregado
);
```

---

## 13. Classificar empregados por faixa salarial

```sql
SELECT nome,
       salario,
       CASE
           WHEN salario > 4000 THEN 'Alto'
           WHEN salario BETWEEN 2000 AND 4000 THEN 'Médio'
           ELSE 'Baixo'
       END AS faixa_salarial
FROM empregado;
```

## 14. Criar tabela `cliente` com CPF único e nome obrigatório

```sql
CREATE TABLE cliente (
    id_cliente INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    nome VARCHAR(50) NOT NULL,
    cpf CHAR(11) UNIQUE NOT NULL,
    uf CHAR(2) NOT NULL
);
```

---

## 15. Criar pedido referenciando cliente com exclusão em cascata

```sql
CREATE TABLE pedido (
    num_pedido INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    id_cliente INTEGER NOT NULL,
    data_pedido DATE NOT NULL,
    FOREIGN KEY (id_cliente)
        REFERENCES cliente(id_cliente)
        ON DELETE CASCADE
);
```

## 17. Atualizar curso de alunos de Ciência da Computação para Informática

```sql
UPDATE aluno
SET curso = 'Informática'
WHERE curso = 'Ciência da Computação';
```

## 18. Remover aluno de matrícula 1

```sql
DELETE FROM aluno
WHERE mat = 1;
```

## 19. Criar view com alunos e disciplinas de carga horária diferente de 60h

```sql
CREATE VIEW vw_aluno_disciplina_not60 AS
SELECT a.nomea, d.nomed
FROM aluno a
INNER JOIN aluno_paga ap ON ap.mat = a.mat
INNER JOIN disc d ON d.d = ap.d
WHERE d.carga_h <> 60;
```

```sql
CREATE INDEX idx_cliente_nome
ON cliente(nome);
```

## 23. Criar transação para registrar pedido e baixar estoque

```sql
BEGIN TRANSACTION;

INSERT INTO pedido (id_cliente, data_pedido)
VALUES (1, CURRENT_DATE);

UPDATE produto
SET valor = valor
WHERE cod_produto = 1;

COMMIT;
```

> Esse exemplo preserva a estrutura transacional, mas em um modelo real haveria uma tabela de estoque/quantidade. O ponto da questão é: as operações relacionadas devem ser confirmadas ou desfeitas juntas.

## 24. Criar trigger que bloqueia cliente sem CPF

```sql
CREATE TRIGGER trg_check_cpf
BEFORE INSERT ON cliente
FOR EACH ROW
BEGIN
    IF NEW.cpf IS NULL THEN
        SIGNAL SQLSTATE '45000'
        SET MESSAGE_TEXT = 'CPF não pode ser nulo';
    END IF;
END;
```

## 25. Corrigir consulta com erro de agregação

Errado:

```sql
SELECT id_gerente, nome, MAX(limite_credito)
FROM empregado
GROUP BY id_gerente;
```

Correção possível:

```sql
SELECT id_gerente, nome, MAX(limite_credito)
FROM empregado
GROUP BY id_gerente, nome;
```

Mas isso muda o nível de agrupamento. Se a intenção era resumir apenas por gerente, remova `nome`:

```sql
SELECT id_gerente, MAX(limite_credito)
FROM empregado
GROUP BY id_gerente;
```

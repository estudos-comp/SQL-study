<h1 align="center">AUTOINCREMENTO POR SGBD</h1>

<i><p align="center">Como gerar chaves automaticamente em Oracle, SQL Server, MySQL e PostgreSQL.</p></i>

---


## Ideia geral

Campo autoincremento é usado quando a chave deve ser gerada automaticamente pelo SGBD. A sintaxe muda entre bancos.

| SGBD | Forma comum |
|---|---|
| Oracle antes do 12c | `SEQUENCE` + uso manual de `nextval`. |
| Oracle 12c+ | `GENERATED ... AS IDENTITY`. |
| SQL Server | `IDENTITY(seed, increment)`. |
| MySQL/MariaDB | `AUTO_INCREMENT`. |
| PostgreSQL antigo | `SERIAL`. |
| PostgreSQL moderno | `GENERATED ... AS IDENTITY`. |

## Oracle antes do 12c: `SEQUENCE`

```sql
CREATE SEQUENCE seq_refeicao_id
START WITH 10
INCREMENT BY 1;
```

Uso:

```sql
INSERT INTO refeicoes
VALUES (seq_refeicao_id.NEXTVAL, DATE '2005-12-07', 204, SYSDATE);
```

`SYSDATE` representa data e horário atuais em Oracle.

## Oracle 12c+: `IDENTITY`

```sql
CREATE TABLE cliente (
    id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    nome VARCHAR2(50)
);
```

Inserções:

```sql
INSERT INTO cliente (nome) VALUES ('Ana');
INSERT INTO cliente (nome) VALUES ('Bruno');
INSERT INTO cliente (nome) VALUES ('Carla');
```

Com `BY DEFAULT`, o banco gera o valor, mas você pode sobrescrever:

```sql
INSERT INTO cliente (id, nome)
VALUES (100, 'Diego');
```

Com `GENERATED ALWAYS AS IDENTITY`, sobrescrever não é permitido sem sintaxe especial.

## SQL Server: `IDENTITY`

```sql
CREATE TABLE mytable (
    col1 INT NOT NULL IDENTITY(1, 100),
    col2 CHAR(5) NOT NULL
);
```

- `1` é a semente: valor inicial.
- `100` é o incremento: quanto aumenta a cada linha.

Exemplo:

```sql
INSERT mytable (col2) VALUES ('Maria');
SELECT @@IDENTITY;
```

`@@IDENTITY` retorna o último valor de identidade inserido na sessão/escopo de execução, mas em projeto real é comum preferir mecanismos mais seguros do SGBD, como `SCOPE_IDENTITY()` no SQL Server.

## MySQL/MariaDB: `AUTO_INCREMENT`

```sql
CREATE TABLE cliente (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nome VARCHAR(50)
);
```

Inserções:

```sql
INSERT INTO cliente (nome) VALUES ('Ana');
INSERT INTO cliente (nome) VALUES ('Bruno');
INSERT INTO cliente (nome) VALUES ('Carla');
```

Definir valor inicial:

```sql
CREATE TABLE cliente (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nome VARCHAR(50)
) AUTO_INCREMENT = 100;
```

Se você informar um valor manual maior:

```sql
INSERT INTO cliente (id, nome) VALUES (200, 'Diego');
INSERT INTO cliente (nome) VALUES ('Elisa');
```

O próximo valor tende a seguir a partir do maior valor já usado.

Resetar contador:

```sql
ALTER TABLE cliente AUTO_INCREMENT = 1;
```

Só funciona corretamente se não houver valores existentes maiores que esse contador.

## PostgreSQL: `SERIAL` e `IDENTITY`

Forma antiga:

```sql
CREATE TABLE professor (
    id_professor SERIAL PRIMARY KEY,
    nome VARCHAR(100) NOT NULL,
    titulacao VARCHAR(50),
    departamento VARCHAR(100)
);
```

Forma moderna, aderente ao padrão:

```sql
CREATE TABLE curso (
    id_curso INT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    nome VARCHAR(100) NOT NULL,
    carga_horaria INT,
    departamento VARCHAR(100)
);
```

Inserção:

```sql
INSERT INTO curso (nome, carga_horaria, departamento)
VALUES ('Engenharia de Software', 3600, 'Tecnologia');
```

## Padrão de prova

Se o enunciado disser “código gerado automaticamente”, escolha a sintaxe do SGBD indicado. Se não indicar SGBD, use uma forma padrão:

```sql
id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY
```

Mas, se o professor estiver usando MySQL nos exemplos, `AUTO_INCREMENT` provavelmente é aceito.
